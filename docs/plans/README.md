# 実装計画書

このディレクトリには、承認された要件定義に基づく実装計画書を格納します。

## 命名規則
`TASK-XXX_feature-name.md`（対応する要件定義書と同じ名前）

例: `TASK-001_agent-auth-flow.md`

## 作成方法
```bash
cp docs/templates/IMPLEMENTATION_PLAN_TEMPLATE.md docs/plans/TASK-XXX_feature-name.md
```

## 前提条件
実装計画書を作成する前に、対応する要件定義書が **Approved** ステータスであることを確認してください。

## ステータス管理
各実装計画書の冒頭にステータスを記載：
- **Planning**: 計画作成中
- **In Progress**: 実装中
- **Review**: レビュー待ち
- **Completed**: 完了

---
承認された実装計画書に基づいて実装を進めてください。
